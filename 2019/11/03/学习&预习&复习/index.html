常用命令

history命令

  • history [n] 显示最后n个历史命令
  • history -c 清空历史命令

ls命令

  • -a : 显示所有文件及目录

  • -C ：按列输出

  • -x：按行输出

  • -l：输出详细信息

  • -t：根据文件修改时间的先后次序列出

查看文件的三个时间stat 命令

三个时间：访问时间、修改时间、状态改动时间

stat [option] filename

查看文件类型file命令

file [option] filename

查找文件或目录 locate

locate [option] 参数

参数：要查找的文件名中含有的字符串

选项 : -i 忽略大小写

💨查找文件内容命令 grep

格式：grep [选项] 匹配字符串 文件列表

  • -i 不区分大小写匹配
  • -v 列出不匹配行
  • -n 显示匹配行号
  • -c 对匹配行进行计数
  • -l列出匹配的文件名称
  • -L列出不匹配的文件名称
  • -H在显示匹配内容前标识符合的文件名称
  • -h在显示匹配内容前不标识符合的文件名称

相关知识

两级提示符

  • 等待命令输入提示符 $， 可以通过修改PS1变量的值来修改该提示符。如：PS1="#"
  • 执行命令时需要输入更多信息>， 可通过修改PS2变量的值来修改。如：PS2="<"

常用通配符

  • ? : 匹配任意单个字符
  • * ：匹配若干个任意字符

文件类型

  • -：普通文件
  • d：目录文件
  • l：链接文件
  • b：块设备文件
  • c：字符设备文件
  • p：管道文件
  • m：共享存储器
  • s：socket文件

文件权限

普通权限：读（r，4）、写（w，2）、执行（x，1）

特殊权限：

  • SUID（Set UID, 4）：当s出现在文件所有者的x权限上时称为SUID，执行者将拥有文件所有者的权限
  • SGID（Set GID, 2）：当s出现在文件所属组的x权限上时称为SGID，执行者在执行过程中会获得程序用户组的支持
  • SBIT（Sticky Bit, 1）：粘滞位，当对某个目录设置了粘滞位权限后，那么该目录中的文件就只能被其所有值执行删除操作了。当设置该权限后，其他用户的x执行权限会显示t（原来有x权限）或T（原来无x权限）

权限设置：chmod [选项] 文件

  • chmod u+x,g+w file0
  • chmod 777 file1 全赋予读写执行权限
  • chmod u=rwx,g=rwx,o=rwx file2全赋予读写执行权限
  • chmod 7777 file3 第一个7是用于设置特殊权限
